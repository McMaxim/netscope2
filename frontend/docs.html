NetScope — a web-based network diagnostics toolkit. Run network utilities from your browser without installing anything.

View on GitHub

Overview

NetScope exposes common network diagnostic tools through a clean REST API and a browser UI. All operations are executed server-side, so they reflect the network conditions and routing from the server's perspective.

📡 Ping

ICMP echo requests with RTT statistics (4 packets, 2s timeout per hop).

🔍 DNS Lookup

Resolves A, AAAA, MX, NS, and TXT records for any domain.

🔌 Port Scanner

TCP connect scan on a custom list of ports (max 20). Checks 22, 80, 443 and more by default.

📋 HTTP Headers

Fetches response headers from any URL, following redirects, showing final URL and status code.

🗺️ Traceroute

Traces the network path to a host, up to 15 hops, 1s timeout per hop.

API Reference

All endpoints accept POST with a JSON body and return JSON. Base URL: /api

POST /api/ping public
Body: {"host": "google.com"}
Returns: {"host", "output": [...lines], "success": bool}
POST /api/dns public
Body: {"host": "github.com"}
Returns: {"host", "records": {"A": [...], "MX": [...], ...}}
POST /api/ports public
Body: {"host": "1.1.1.1", "ports": [80, 443, 22]} (max 20 ports)
Returns: {"host", "results": [{"port": 80, "open": true}, ...]}
POST /api/headers public
Body: {"url": "https://example.com"}
Returns: {"url", "status": 200, "headers": {...}}
POST /api/traceroute public
Body: {"host": "8.8.8.8"}
Returns: {"host", "hops": [...lines]}
GET /api/health healthcheck
Returns: {"status": "ok"}

Self-Hosting

NetScope is fully Dockerized. You need Docker, Docker Compose, and a domain pointing to your server.

git clone https://github.com/McMaxim/netscope.git
cd netscope

# Edit Caddyfile — replace yourdomain.com with your domain
nano Caddyfile

docker compose up -d

Caddy handles TLS certificates automatically via Let's Encrypt. No extra configuration needed.

Architecture

Browser
  └─▶ Caddy (HTTPS :443)
        ├─▶ /api/* ──▶ FastAPI (uvicorn :8000)
        └─▶ /*    ──▶ Static files (frontend/)

All network operations run inside the backend container. Ping and traceroute use NET_RAW capabilities. Port scan uses async TCP connect with a 2s timeout per port.

Security Notes

Host input is validated with /^[a-zA-Z0-9.\-_]+$/ to prevent shell injection. All subprocess calls use exec form (no shell). Port scan is limited to 20 ports per request. Timeouts are enforced on every operation.
